Usage

Operating this application is simple. There are basically three major parts: the Home/Display page, the File Management page, and the Help Page, where you are at.

Home Page

The home page displays the current state of the stickman. In this page you can switch out different parts of the displayed stickman. For example, clicking the pirate hat button will immediately change out the regular stickman head for a pirate hat. If you select the buttons 'Default', 'Pirate', or 'Leprechaun' it will switch out the entire stickman to the predetermined figure.

Do not worry if you accidentally changed a part of the stickman you did not want to. You can simply use the undo and redo buttons in the upper right hand corner of the stickman area to revert to any previous state. The save button is right next to the undo/redo buttons and will allow you to save the current stickman to the server for later access. You can access this saved file in the File Management Page...

File Management Page

In the File Management page, you can look to see what character files are available to load. You can also upload as many of your own files that you want (files with the same name are overwritten). The Download link next to each file, allows you to see what a file has in it, as well as, save it to your own computer if you wish. All files displayed have a Load option which will take you back to the Home Page with the stickman already having the file loaded into it. The Delete option next to files just removes that file.

Files

In order to successfully upload and load your files they must be a txt file in the following format:

0
2
1
1
1

A 0 refers to all images that are part of the Default stickman. 1 refers to all images that are part of the Pirate stickman. 2 refers to all images that are part of the Leprechaun stickman. Line 1 is the head, line 2 is left-arm, line 3 is body, line 4 is right-arm, and line 5 is legs. File are also saved in this format.

Help Page

Finally, the help page that you are on right now, gives a brief description of the project as well as lists all requirements, extensions, and CSS rules.

Limitations

The only limitation a user might experience is that any kind of page refresh will destroy any previous history of the stickman, and therefore undo/redo will not work.
